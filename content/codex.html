① 슬래시 명령어 (/) — 카테고리별 + 예시

채팅창에 / 입력. 클로드와 겹치는 게 많음.

명령뜻 쉬운 예시

② AGENTS.md & 설정

AGENTS.md

코덱스의 규칙 파일. 클로드의 CLAUDE.md와 같은 역할. Agent Skills 오픈표준이라 여러 AI 도구가 공유.

읽는 순서 (병합): 프로젝트 루트(보통 Git 루트)부터 현재 폴더까지 내려가며 AGENTS.override.md AGENTS.md 순으로 찾아 전부 이어붙임. 현재 폴더에 가까운 파일이 나중에 와서 우선(앞 규칙 덮어씀).
쉽게: 회사 전체 규칙(루트 AGENTS.md) + 우리 팀 규칙(하위폴더 AGENTS.md)이 합쳐지고, 더 가까운 팀 규칙이 이김. 위아래 누적.

config.toml

~/.codex/config.toml기본 설정(모델·승인·MCP 등)
codex -c key=value그 실행만 임시 덮어쓰기
profiles설정 묶음 전환(작업별 프로파일)
쉽게: 클로드의 settings.json에 해당. 단 코덱스는 .toml 형식.

③ 승인 모드 (Approval)

코덱스가 얼마나 알아서 하느냐. 안전장치.

모드뜻언제
Auto (기본)작업 폴더 안에선 알아서일반 개발 — 폴더 밖은 물어봄
Read-only읽기만, 수정은 자문코드 분석·리뷰만 할 때(안전)
Full Access제한 없음신뢰 환경·자동화 ( 위험)
쉽게: 신입에게 주는 권한 등급. 견습(읽기만)정직원(폴더 안 자유)팀장(다 가능). 위험할수록 사람 확인.

⑤ 파일 읽는 흐름 (클로드/코덱스 공통)

코덱스도 클로드처럼 규칙파일 먼저 읽고 작업 파일 read/write. 다른 건 파일 이름·설정 형식뿐.

시작 — 루트~현재폴더 AGENTS.md 전부 읽어 합침(가까운 게 우선)
요청 코덱스가 승인모드 확인(Auto/Read-only/Full)
read 관련 파일 (/mention·/ide로 콕 집기 가능)
처리
write 코드 — 위험 작업은 /approve 거쳐 승인
핵심: 하네스 설계 원리(고정 IO·승인게이트·컨텍스트 격리)는 클로드든 코덱스든 동일. 도구만 갈아끼우면 됨. ( 클로드코드 탭 ⑦ 예시 하네스 참고)
