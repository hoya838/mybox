① 왜 AI가 나왔나

사람이 규칙을 코드로 다 못 씀(고양이 인식·스팸 필터) "데이터 주면 컴퓨터가 규칙 스스로 찾게" = 머신러닝.

단계핵심
규칙 기반사람이 if문으로 직접 — 예외 많으면 무너짐
머신러닝(ML)데이터+정답 규칙(모델) 학습. 단 특징은 사람이 정함
딥러닝(DL)신경망이 특징 추출까지 자동. 2012 알렉스넷 폭발(데이터+GPU+오픈소스)

② LLM 핵심 용어

용어뜻쉽게
토큰LLM이 처리하는 텍스트 조각"안녕"도 토큰 2개. 비용·용량 결정 단위
컨텍스트 윈도우한 번에 처리하는 정보량AI의 작업 책상 크기
환각(할루시네이션)모르면서 아는 척 틀린 답학습데이터에 없는 걸 물으면 발생
파인튜닝특정 용도로 추가 학습기본기 위에 전문 교육
예문: "이번 주 미팅 일정은?" LLM은 모름(학습데이터에 없음) 환각. 해결 = 캘린더를 같이 줌(RAG).

③ RAG (검색 증강 생성)

AI가 답하기 전 관련 자료를 먼저 찾아보고 답하는 방식. 환각·지식단절 해결.

RAG 6단계 (배민 사례): ① 요구분석 ② 프레임워크 결정 ③ 청킹(문서 쪼개기) ④ 임베딩(숫자 변환) ⑤ 저장(벡터DB) ⑥ 검색답변
쉽게: 시험 볼 때 책 통째 못 외우니까 문제 보고 관련 페이지만 펴서 답하는 것. 그게 RAG.
RAG 주의: 단순히 문서 자르면 앞뒤 맥락 사라져 부정확. 실용책 = 하이브리드 검색(의미+키워드) + 컨텍스추얼 청킹(조각에 위치 설명 부착, 실패 67%) + 요약본 검색원문 전달.

④ AI 에이전트

대답만 하는 챗봇을 넘어 인식판단행동(실행)까지 하는 AI.

구성 = 모델(판단) + 도구(웹검색·캘린더 등 행동수단) + 오케스트레이션(흐름 조율) 루프로 반복.
예문: "미팅 잡아줘" 에이전트가 일정 확인 + 캘린더 등록 + 자료 작성 + 메일 전송까지 자동.
