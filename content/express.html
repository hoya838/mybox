만드는 것: FastAPI와 같은 API를 JS로. 비교하며 두 언어 체화.

핵심 개념

1. 라우트 — URL과 핸들러

import express from 'express';
const app = express();
app.use(express.json());        // JSON 본문 파싱

app.get('/todos', (req, res) => {
  res.json([{ title: '공부' }]);
});
app.post('/todos', (req, res) => {
  const todo = req.body;        // 받은 데이터
  res.status(201).json(todo);
});
app.listen(3000);
쉽게: app.get('/주소', 함수) = FastAPI @app.get과 같은 역할. req=요청, res=응답.

2. 미들웨어 — 중간 처리

app.use((req, res, next) => {
  console.log(req.method, req.url);  // 모든 요청 로그
  next();    // 다음 단계로 넘김
});
쉽게: 요청이 거쳐 가는 검문소. 로그·인증·검증을 중간에. next()=통과.

3. DB 연동 + 라우터 분리

// PostgreSQL 연결 (pg 또는 Prisma)
import pg from 'pg';
const db = new pg.Pool({ connectionString: process.env.DB_URL });
const { rows } = await db.query('SELECT * FROM todos');
쉽게: pg로 PostgreSQL에 SQL 날림. 커지면 라우트를 파일별로 분리(MVC).

쉽게: 둘이 거의 같은 일. FastAPI가 검증·문서 자동이라 편하고, Express는 JS라 프론트와 언어 통일.

필수 패턴 — 줄별 주석

Express = Node.js 위 최소 웹 프레임워크. 미들웨어(요청이 거치는 함수 사슬) + 라우팅이 핵심. 주석 = 그 줄이 하는 일.

// ── 앱 만들기 ───────────────────────────────────
const express = require("express");  // 프레임워크 불러오기(CommonJS)
const app = express();               // 앱 객체 생성

// ── 미들웨어: 모든 요청이 순서대로 거침 ─────────
app.use(express.json());             // 요청 본문(JSON)을 req.body로 파싱해줌(필수)
app.use((req, res, next) => {        // 커스텀 미들웨어(로깅 등)
  console.log(req.method, req.url);  // 요청 메서드·경로 기록
  next();                            // next()=다음 미들웨어로 넘김(안 부르면 멈춤)
});

// ── 라우트: 메서드 + 경로 + 핸들러 ──────────────
app.get("/posts", (req, res) => {    // GET /posts
  res.json([{ id: 1, title: "hi" }]); // res.json() = JSON 응답 전송
});

app.get("/posts/:id", (req, res) => {  // :id = 경로 변수
  const id = req.params.id;          // req.params로 꺼냄("/posts/3" → "3")
  res.json({ id });
});

app.post("/posts", (req, res) => {   // POST /posts
  const { title } = req.body;        // 본문에서 title 꺼내기(express.json 덕분)
  if (!title) return res.status(400).json({ error: "title 필요" });
  //  검증 실패 시 400(잘못된 요청). return으로 함수 종료
  res.status(201).json({ id: 1, title }); // 201=생성됨
});

// ── 에러 핸들러: 인자 4개짜리(맨 뒤에 위치) ─────
app.use((err, req, res, next) => {   // err가 첫 인자면 에러 전용 미들웨어
  res.status(500).json({ error: err.message }); // 500=서버 에러
});

// ── 서버 켜기 ───────────────────────────────────
app.listen(8000, () => console.log("http://localhost:8000")); // 8000 포트 대기
꼭 알아야 할 지식: ①미들웨어는 순서대로 실행 — express.json()을 라우트보다 위에 둬야 req.body가 참 ②next() 안 부르면 요청이 그 자리에서 멈춤 ③경로 :id=req.params, 쿼리 ?q==req.query, 본문=req.body ④에러 미들웨어는 인자 4개(err 포함)로 구분, 맨 마지막에 ⑤검증·문서는 수동(FastAPI와 차이) — zod·express-validator 별도 ⑥res는 한 번만 응답(두 번 보내면 에러).
