Node.js 백엔드 · JS 런타임

JavaScript를 서버에서 실행. 프론트와 같은 언어로 백.

1
기초 모듈(import), npm, 파일시스템(fs), 비동기·콜백, 환경변수
2
중급 Promise·async/await, 이벤트 emitter, 스트림, package.json
3
심화 이벤트 루프 심화, 클러스터, 성능·메모리, 테스트(Jest)
import http from 'http';
http.createServer((req, res) => {
  res.end('Hello from Node');   // 가장 단순한 서버
}).listen(3000);
만드는 것: JS 서버 기반. 실무는 Express 얹어 편하게.

핵심 개념

1. 모듈 + npm

npm init -y                 # 프로젝트 시작
npm install express         # 패키지 설치
import express from 'express';   // 가져오기 (ESM)
쉽게: npm = 앱스토어. 필요한 도구 설치. package.json에 목록 기록됨.

2. 비동기 — JS와 동일

import fs from 'fs/promises';
const data = await fs.readFile('a.txt', 'utf-8');  // 기다림
쉽게: 브라우저 JS의 async/await 그대로. 파일·DB·네트워크는 다 비동기.

3. 환경변수

const port = process.env.PORT || 3000;   // 환경변수 or 기본값
// .env 파일에 비밀키·DB주소 보관 (git에 안 올림)
쉽게: 비밀번호·설정은 코드에 박지 말고 .env에. 환경마다 다르게.

필수 패턴 — 줄별 주석

Node.js는 언어가 아니라 JS 실행환경(런타임). 브라우저 밖에서 파일·서버·프로세스를 다룬다. 주석 = 그 줄이 하는 일.

// ── 1. 모듈: 코드 나눠 쓰기 ─────────────────────
const fs = require("fs");            // Node 내장 모듈 불러오기(CommonJS 방식)
// import fs from "fs";              // ESM 방식(package.json에 "type":"module"일 때)
const { add } = require("./math");   // 내가 만든 파일 가져오기(.js 생략 가능)
module.exports = { add };            // 이 파일 것을 남에게 내주기

// ── 2. 파일 읽고 쓰기(fs) ───────────────────────
const data = fs.readFileSync("a.txt", "utf-8"); // 파일 통째로 읽기(동기=끝날 때까지 대기)
fs.writeFileSync("b.txt", "내용");   // 파일 쓰기(없으면 생성)

// ── 3. 환경변수: 비밀·설정 분리 ─────────────────
const port = process.env.PORT || 3000; // process.env로 .env 값 읽기, 없으면 3000
// process = 실행 중인 Node 프로세스 정보(인자·환경·종료 등)

// ── 4. 비동기: 안 기다리고 넘어감 ───────────────
const fsp = require("fs/promises");  // 프로미스 버전 fs
async function readAll() {
  const txt = await fsp.readFile("a.txt", "utf-8"); // await=결과 올 때까지 대기
  return txt;                        // 동기(readFileSync)와 달리 서버를 안 멈춤
}

// ── 5. 내장 HTTP 서버(프레임워크 없이) ──────────
const http = require("http");        // 내장 http 모듈
http.createServer((req, res) => {    // 요청마다 이 함수 실행
  res.writeHead(200, { "Content-Type": "application/json" }); // 상태·헤더
  res.end(JSON.stringify({ ok: true })); // 응답 본문 전송하고 종료
}).listen(port);                     // port에서 요청 대기(보통은 Express로 대체)

// ── 6. npm: 패키지·스크립트 ─────────────────────
// npm init -y            → package.json 생성
// npm install express    → 외부 패키지 설치(node_modules에)
// npm run dev            → package.json scripts의 dev 실행
꼭 알아야 할 지식: ①Node는 런타임 — 언어는 그대로 JS, 브라우저 API(document 등) 없고 대신 fs·http·process ②모듈 2방식: require(CommonJS)·import(ESM) — 섞으면 에러 ③동기(...Sync) 남용 금지 — 서버가 그 동안 멈춤. 비동기 await 기본 ④process.env + .env로 비밀 분리, .gitignore에 .env 필수 ⑤package.json=프로젝트 명세(의존성·스크립트), node_modules는 git에 안 올림 ⑥실무 서버는 http 직접 대신 Express.
