웹개발 — 프론트·백엔드·클라우드·API

필수 용어 + 예문 + 쉬운 해석 + 언어 정리.

① 프론트엔드 (Front-end)

사용자가 직접 보고 클릭하는 화면. 브라우저에서 돌아감.

기술역할쉽게
HTML화면 뼈대·내용집의 골조·벽
CSS꾸미기(색·폰트·배치)페인트·인테리어
JavaScript동작(버튼·이벤트)전기·움직이는 장치
React·Vue·Next화면을 효율적으로 만드는 도구조립식 가구 키트
예문: 쇼핑몰 "장바구니 담기" 버튼을 눌렀을 때 숫자가 +1 되는 것 = JavaScript가 처리한 이벤트.
쉽게: React는 사실 "프레임워크"처럼 보여도 갖다 붙이는 라이브러리에 가깝다. 왜 생겼는지 알면 외울 필요 없음.

② 백엔드 (Back-end)

화면 뒤에서 데이터를 처리·저장하는 엔진. 서버에서 돌아감.

탄생 이유 = 동적 생성. 옛날엔 서버가 HTML 파일을 그냥 꺼내줬는데, 방명록·검색은 "누가 뭘 남길지" 미리 못 만듦 요청 올 때 그 자리에서 만들어야 했다(1993 CGIPHP…).
구성역할
서버요청 받고 응답하는 컴퓨터(24시간 켜짐)
WAS웹 애플리케이션 서버 — 로직 실행
언어/프레임워크Node.js·Python(Django)·Java(Spring)·PHP
예문: 로그인 = 프론트가 보낸 아이디/비번을 백엔드가 받아 DB의 회원 정보와 대조 맞으면 "통과" 응답.

③ API / REST

프론트와 백엔드가 데이터 주고받는 약속(메뉴판). "이렇게 요청하면 이렇게 응답한다."

REST = CRUD (가장 널리 쓰는 형식)
동작뜻HTTP 메서드예
Create생성POST새 글 등록
Read조회GET글 목록 보기
Update수정PUT/PATCH글 고치기
Delete삭제DELETE글 지우기
쉽게: API = 식당 점원. 손님(프론트)이 "김치찌개요"(요청) 점원이 주방(DB)에서 받아 "여기요"(응답). 손님이 주방 직접 안 들어감.
예문: GET /books/3 = "3번 책 정보 줘" 응답 200 OK + 책 데이터(JSON).

④ 데이터베이스 (DB)

데이터를 구조적으로 저장·관리하는 곳. DB=집, 테이블=방, 행/열=엑셀 시트.

용어쉽게
DB큰 창고(집)
테이블종류별 방(회원방·상품방)
SQLDB에 명령하는 언어("회원 다 줘")
스토리지파일(이미지·동영상) 저장 — DB와 별개(S3 등)

⑤ 클라우드 / 배포

만든 걸 인터넷에 올려 남도 쓰게 하는 것.

용어뜻비유
서버요청에 응답하는 컴퓨터24시간 켜진 가게
IP인터넷상 컴퓨터 주소(숫자)집 주소
도메인IP를 사람이 읽는 이름(naver.com)가게 간판
DNS도메인IP 변환주소록
클라우드서버를 빌려 쓰는 서비스(AWS 등)전세 대신 월세
도커실행 환경 통째 포장이사 박스
예문: localhost:3000 = 내 컴퓨터에서만 열림. 친구도 보게 하려면 클라우드 서버에 배포 + 도메인 연결.

⑥ 언어 자료 정리

언어주 용도한 줄
JavaScript프론트+백(Node)웹의 공용어. 화면 동작 필수
TypeScriptJS + 타입JS에 안전장치(오타·타입 오류 잡음)
Python백엔드·AI·데이터쉽고 AI 생태계 최강
Java대기업 백엔드Spring으로 견고한 서버
SQLDB 조회데이터 다루는 필수 언어
HTML/CSS화면 구조·스타일엄밀히 언어 아닌 마크업, 그래도 필수
쉽게: 언어 = 컴퓨터와 소통하는 외국어. 사람마다 영어·중국어 쓰듯, 상황 따라 언어 고름. 입문은 JavaScript 또는 Python 하나로 시작 추천.
