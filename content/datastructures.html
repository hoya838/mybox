자료구조 — 데이터를 어떻게 담을까

코딩테스트 필수 + 개발 사고력의 뼈대. C 기반 개념이지만 JS·Python으로 바로 쓸 수 있다.

왜 배우나: 자료구조 = 데이터를 저장하는 방식. 어떤 그릇에 담느냐에 따라 꺼내는 속도·순서가 달라진다. 실무에서 직접 구현할 일은 적지만, 어떤 구조가 왜 빠른지 모르면 성능 문제가 생겼을 때 원인을 못 찾는다. 코딩테스트에선 직접 씀.

① 배열 (Array)

가장 기본 — 번호 붙은 칸에 순서대로 담기.

특징: 인덱스로 바로 접근 → 조회 O(1). 삽입·삭제는 뒤를 밀어야 해서 느림 O(n).
비유: 번호 매긴 사물함. 3번 사물함 바로 열 수 있음. 근데 2번 뺄 때 3·4·5번을 한 칸씩 당겨야 함.
언어코드
JavaScriptconst arr = [1, 2, 3];
arr[0]; // 1, O(1)
Pythonarr = [1, 2, 3]
arr[0] # 1, O(1)
쓰는 때: 순서 있고, 자주 조회하고, 삽입·삭제가 드물 때.

② 스택 (Stack)

나중에 넣은 게 먼저 나온다 — LIFO (Last In, First Out).

비유: 접시 쌓기. 맨 위 접시부터 꺼냄. 아래 꺼내려면 위를 다 빼야 함.
연산 2개: push(추가) / pop(꺼내기) — 둘 다 O(1).
실생활 쓰임이유
브라우저 뒤로가기방문 기록을 스택에 쌓고 pop
함수 콜스택함수 호출 순서 추적 (재귀)
Ctrl+Z 실행취소변경 이력 스택에 쌓고 pop
괄호 검증여는 괄호 push, 닫는 괄호에 pop 비교
언어코드
JavaScriptconst stack = [];
stack.push(1); stack.push(2);
stack.pop(); // 2
Pythonstack = []
stack.append(1); stack.append(2)
stack.pop() # 2

③ 큐 (Queue)

먼저 넣은 게 먼저 나온다 — FIFO (First In, First Out).

비유: 줄서기(대기열). 먼저 온 사람이 먼저 처리됨.
연산 2개: enqueue(뒤에 추가) / dequeue(앞에서 꺼내기).
실생활 쓰임이유
프린터 인쇄 대기먼저 보낸 문서부터 인쇄
서버 요청 처리먼저 온 요청부터 처리
너비 우선 탐색(BFS)그래프 탐색 알고리즘 핵심
언어코드
JavaScriptconst queue = [];
queue.push(1); queue.push(2);
queue.shift(); // 1 (앞에서 꺼냄)
Pythonfrom collections import deque
q = deque()
q.append(1); q.append(2)
q.popleft() # 1
Python 팁: 일반 리스트로 큐 쓰면 popleft()가 O(n). collections.deque 쓰면 O(1).

④ 해시 (Hash / 딕셔너리)

키(key)로 값(value)을 바로 꺼낸다 — 조회 O(1).

비유: 사전(dictionary). "apple"을 찾으면 바로 뜻이 나옴. 처음부터 뒤질 필요 없음.
실제론: 키를 해시 함수로 변환 → 배열 인덱스로 저장. 그래서 조회가 O(1).
실생활 쓰임이유
로그인 세션 관리user_id → 세션 데이터 빠른 조회
중복 제거set도 해시 기반 — 이미 있는지 O(1) 확인
빈도 카운트단어 → 등장 횟수 매핑
언어코드
JavaScriptconst map = { name: "홍길동", age: 20 };
map["name"]; // "홍길동", O(1)
Pythond = {"name": "홍길동", "age": 20}
d["name"] # "홍길동", O(1)

⑤ 트리 (Tree)

계층 구조 — 부모·자식 관계로 뻗어나가는 구조.

비유: 회사 조직도, 파일 시스템(폴더 안에 폴더).
구성: 루트(최상단) → 부모 → 자식 노드. 사이클(순환) 없음.
종류특징쓰임
이진 트리자식 최대 2개기반 구조
이진 탐색 트리(BST)왼쪽<부모<오른쪽 규칙 → 탐색 O(log n)정렬된 데이터 검색
DOM 트리HTML 요소 계층브라우저가 화면 렌더링
쉽게: 탐색이 O(log n) — 100만 개 데이터도 20번이면 찾음. 배열 순차탐색(O(n))과 차원이 다름.

⑥ 그래프 (Graph)

노드(점)와 간선(선)으로 관계를 표현 — 트리의 상위 개념.

비유: 지하철 노선도(역=노드, 노선=간선), SNS 친구 관계.
트리와 차이: 그래프는 사이클(순환) 가능, 방향 없을 수도 있음.
탐색 방법도구언제
BFS (너비 우선)큐최단 경로, 가까운 것부터
DFS (깊이 우선)스택 or 재귀경로 탐색, 미로 풀기
쓰임: 지도 길찾기·SNS 친구추천·인터넷 라우팅·의존성 관리(npm 패키지 충돌 탐지).
