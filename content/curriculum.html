1. JS 생태계 포함관계 — 뭐가 뭘 감싸나

이 기술들은 따로따로가 아니라 층층이 쌓이는 구조다. 아래가 위를 받쳐준다.

전체 지도
JavaScript (언어 — 가장 기반)
├── TypeScript ← JS에 타입 추가한 상위집합. JS 파일 그대로 TS에서 동작
│
├── [브라우저 환경 — 프론트엔드]
│ ├── 바닐라 JS (순수, 라이브러리 없음)
│ ├── React ← JS(또는 TS) 위에 올라가는 UI 라이브러리
│ └── Next.js ← React 위에 올라가는 프레임워크 (SSR·파일기반라우팅 추가)
│
└── [서버 환경 — 백엔드]
    ├── Node.js ← JS를 서버(브라우저 밖)에서 실행하는 런타임
    └── Express ← Node.js 위에 올라가는 웹 프레임워크
기술종류무엇 위에?한 줄
JavaScript언어— (기반)모든 것의 뿌리. 브라우저·서버 둘 다 동작
TypeScript언어 (JS 상위집합)JavaScript 포함JS에 타입 안전장치 추가. JS 코드 그대로 TS에서 돌아감
Node.js런타임 (실행환경)JavaScript"JS는 원래 브라우저 전용" → Node가 서버에서 실행 가능하게 함
React라이브러리JavaScript (또는 TS)UI를 컴포넌트 단위로 만드는 도구. 프레임워크 아님
Next.js프레임워크ReactReact에 SSR·파일기반 라우팅·API 라우트 추가. React만으론 부족한 것들을 채움
Express프레임워크Node.jsNode.js로 API 서버 만드는 틀. 라우팅·미들웨어 제공
핵심 포인트
  • TypeScript ⊃ JavaScript — TS는 JS를 포함한다. JS 파일을 TS 프로젝트에 그냥 넣어도 동작. "JS에 타입 레이어 추가" = TS
  • Node.js ≠ 언어 — Node는 언어가 아니라 실행 환경(런타임). "JS를 서버에서 돌려주는 엔진"
  • React ≠ 프레임워크 — React는 라이브러리. UI만 담당. 라우팅·서버·배포는 React 밖
  • Next.js = React + α — Next는 React를 포함하고 SSR·라우팅·API·이미지최적화 추가. "React로 할 수 있는 것 + 못하는 것까지"
비유로: JavaScript = 재료(밀가루). TypeScript = 밀가루+계량컵(오차 방지). Node.js = 오븐(밀가루를 서버에서 구울 수 있게). React = 제빵 키트(쉽게 빵 모양 만들기). Next.js = 제빵 키트+배달포장+레시피북(빵 만들기+배달까지).

5. 권장 학습 순서

1
HTML·CSS·JS (프론트 기초) — 화면 먼저 만들어 보기
2
Python 기초 — 쉬운 언어로 프로그래밍 감 잡기
3
PostgreSQL + SQL — 데이터 다루기
4
FastAPI — Python으로 첫 API 서버 (게시판 CRUD 만들기)
5
Node.js + Express — 같은 게시판을 JS로 다시 (비교 학습)
6
TypeScript → React — 프론트 본격화, 만든 API와 연결
핵심: 각 단계마다 작은 프로젝트로 직접. "게시판 CRUD"를 FastAPI로 한 번, Express로 한 번 만들면 백엔드 핵심(라우팅·DB·인증)이 두 언어로 체화됨. (vault "생존법": 하나 깊게 → 둘째는 빠름)
