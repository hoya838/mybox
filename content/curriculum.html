Python백엔드서버·AI·데이터

쉽게: FastAPI·Express = "서버 만드는 틀"(백). PostgreSQL = "데이터 창고"(백). React = "화면 조립 키트"(프론트). 프레임워크는 언어 위에 얹는 골격이다.
내 백엔드 2갈래: Python → FastAPI로 서버 / JavaScript → Node.js + Express로 서버. 둘 다 PostgreSQL에 연결. 같은 일을 두 언어로 — 비교하며 배우면 개념이 또렷해짐.

1. 기술 전체 지도 — 화면·서버·데이터로 나누면

이 기술들은 각자 담당 구역이 다르다. 구역부터 잡으면 관계가 보인다.

🖥 화면 (프론트엔드)
HTML — 뼈대
CSS — 스타일
JavaScript — 동작
TypeScript — JS + 타입
React — 화면 조립 도구
Next.js — React + 서버렌더
⚙️ 서버 (백엔드)
Node.js — JS 서버 실행환경
Express — Node API 프레임워크
Python — 서버 언어
FastAPI — Python API 프레임워크
🗄 데이터 (DB)
PostgreSQL — 데이터 저장소

프론트엔드 안에서 층층이 쌓이는 구조

HTML·CSS·JS는 기본 재료, 그 위에 도구들이 올라간다.

① 기본 재료
  HTML — 화면 뼈대 ("여기 제목, 여기 버튼")
  CSS — 색·크기·배치 꾸미기
  JavaScript — 버튼 클릭, 데이터 불러오기 등 동작

② JS에 안전장치 추가
  TypeScript — JS 그대로인데 타입 오류를 미리 잡아줌. "JS의 업그레이드판"
    → JS 코드 = TS에서 그대로 동작. TS ⊃ JS

③ JS로 화면을 더 쉽게 만드는 도구
  React — 화면을 작은 부품(컴포넌트)으로 나눠 조립. 상태(state) 바뀌면 자동 갱신
    → JS(또는 TS) 위에 올라가는 라이브러리

④ React에 서버 기능 추가
  Next.js — React로 만든 화면을 서버에서 미리 렌더링(SSR). 라우팅·API도 포함
    → React 위에 올라가는 프레임워크. "React가 못 하는 것까지"

백엔드 — JS가 서버로 가는 방법

문제: JS는 원래 브라우저 전용 언어
  → 서버에서 실행하려면 실행환경이 필요

Node.js — "JS를 서버에서 돌려주는 엔진(런타임)". 언어가 아님
  → Node 덕분에 JS 하나로 프론트+백 모두 가능해짐

Express — Node.js 위에 올라가는 웹 프레임워크
  → "이 URL로 요청 오면 이렇게 응답해라" 라우팅·미들웨어 제공
질문답
TypeScript랑 JavaScript 뭐가 달라?TS = JS + 타입 체크. JS 코드 그대로 TS에서 동작. 오타·타입 오류를 실행 전에 잡음
Node.js가 언어야?아님. JS를 서버에서 실행하는 환경(런타임). 언어는 그대로 JavaScript
React랑 Next.js 뭐가 달라?React = 화면만. Next = React + SSR + 라우팅 + API. React 위에 올라감
React가 프레임워크야?아님. 라이브러리. UI만 담당. 라우팅·서버는 React 밖(Next.js가 해결)
Express랑 FastAPI 차이?언어만 다름. Express = Node.js(JS)로 서버. FastAPI = Python으로 서버. 역할 같음

DB 테이블 구조 — posts

컬럼타입설명

글 목록 불러오기 — 4단계 전체 흐름

1
프론트 (devlog.kr) — 백엔드에 목록 요청
게시판 페이지가 열리는 순간 api.devlog.kr 서버로 GET 요청 
// devlog.kr — 게시판 페이지 컴포넌트 (Next.js)
const [posts, setPosts] = useState([]);  // 글 목록 저장할 빈 배열

useEffect(() => {
  // 컴포넌트가 화면에 뜨자마자 실행
  fetch('https://api.devlog.kr/posts')   // 백엔드 서버 주소로 HTTP GET 요청
    .then(res => res.json())             // 응답을 JSON으로 변환
    .then(data => setPosts(data));       // 받은 글 목록을 posts에 저장 → 화면 자동 갱신
}, []);  // [] = 처음 한 번만 실행
2
백엔드 (api.devlog.kr) — 요청 받아 DB 조회
Express가 요청을 받고 PostgreSQL에 SQL을 전송 
// ── DB 연결 (서버 시작 시 한 번만 설정) ─────────────────────
const { Pool } = require('pg');   // pg = Node.js ↔ PostgreSQL 연결 드라이버 (npm install pg)

const db = new Pool({
  host: process.env.DB_HOST,      // 환경변수로 숨김 (예: db.supabase.com)
  port: 5432,                     // PostgreSQL 기본 포트
  database: 'devlog',             // DB 이름
  user: process.env.DB_USER,      // 환경변수로 숨김
  password: process.env.DB_PASS,  // 환경변수로 숨김 (코드에 비번 직접 쓰면 안 됨)
});
// db.query() 호출 시 → pg 드라이버가 PostgreSQL에 네트워크로 SQL 전달

// ── 라우터 — GET /posts ──────────────────────────────────
app.get('/posts', async (req, res) => {
  // GET https://api.devlog.kr/posts 요청이 오면 이 함수 실행
  const result = await db.query(`
    SELECT id, title, author, created_at
    FROM posts
    ORDER BY created_at DESC   -- 최신 글이 위에 오도록 정렬
    LIMIT 20                   -- 한 번에 최대 20개만
  `);
  // result.rows = PostgreSQL이 돌려준 행(row) 배열
  // [
  //   { id: 3, title: "Next.js 배포 후기", author: "jun", created_at: "2026-07-04..." },
  //   { id: 2, title: "Express 시작하기",  author: "hoya", created_at: "2026-07-03..." },
  //   ...
  // ]
  res.json(result.rows);   // 그 배열을 JSON으로 변환해 프론트에 응답
});
pg 드라이버가 하는 일: Node.js(JS)는 PostgreSQL(C++)과 직접 대화 불가. pg 드라이버가 중간에서 통역 — SQL 문자열을 PostgreSQL이 이해하는 프로토콜로 변환해 전송, 결과를 JS 객체 배열로 변환해 돌려줌.
3
DB (devlog PostgreSQL) — SQL 실행 후 데이터 반환
posts 테이블에서 조건에 맞는 행을 꺼내 Express에 돌려줌 
-- PostgreSQL이 실행하는 SQL (Express가 보낸 것)
SELECT id, title, author, created_at
FROM posts
ORDER BY created_at DESC
LIMIT 20;

-- 결과 (rows):
-- id | title                  | author | created_at
-- ---+------------------------+--------+---------------------
--  3 | Next.js 배포 후기      | jun    | 2026-07-04 10:23:00
--  2 | Express 시작하기       | hoya   | 2026-07-03 09:11:00
--  1 | PostgreSQL 첫 설정     | jun    | 2026-07-02 14:05:00
4
프론트 — 받은 데이터로 화면 렌더링
JSON 배열을 받아 게시판 목록 UI로 표시 
// devlog.kr 게시판 목록 화면
return (
  <div>
    <h1>devlog 게시판</h1>
    {posts.map(post => (               // posts 배열을 하나씩 꺼내 카드로 표시
      <div key={post.id}>             // key = React가 각 항목 구분하는 고유값
        <h2>{post.title}</h2>        {/* "Next.js 배포 후기" */}
        <span>{post.author}</span>   {/* "jun" */}
      </div>
    ))}
  </div>
);

CRUD 전체 — devlog.kr 기준

동작프론트 요청ExpressPostgreSQL
글 목록 보기GET api.devlog.kr/postsapp.get('/posts')SELECT ... FROM posts
글 하나 보기GET api.devlog.kr/posts/3app.get('/posts/:id')SELECT ... WHERE id=3
글 쓰기POST api.devlog.kr/posts + {title, content}app.post('/posts')INSERT INTO posts ...
글 수정PUT api.devlog.kr/posts/3 + {title}app.put('/posts/:id')UPDATE posts SET ... WHERE id=3
글 삭제DELETE api.devlog.kr/posts/3app.delete('/posts/:id')DELETE FROM posts WHERE id=3
핵심: 프론트(devlog.kr)는 화면만. 데이터 요청은 항상 백엔드(api.devlog.kr)에 HTTP로. 백엔드는 받아서 DB에 SQL 실행 후 JSON 응답. 프론트는 절대 DB에 직접 접근하지 않는다 — 보안 때문.

5. 권장 학습 순서

1
HTML·CSS·JS (프론트 기초) — 화면 먼저 만들어 보기
2
Python 기초 — 쉬운 언어로 프로그래밍 감 잡기
3
PostgreSQL + SQL — 데이터 다루기
4
FastAPI — Python으로 첫 API 서버 (게시판 CRUD 만들기)
5
Node.js + Express — 같은 게시판을 JS로 다시 (비교 학습)
6
TypeScript → React — 프론트 본격화, 만든 API와 연결
핵심: 각 단계마다 작은 프로젝트로 직접. "게시판 CRUD"를 FastAPI로 한 번, Express로 한 번 만들면 백엔드 핵심(라우팅·DB·인증)이 두 언어로 체화됨. (vault "생존법": 하나 깊게 → 둘째는 빠름)
