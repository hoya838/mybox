개발자 로드맵 — 입문부터 심화까지

"코딩 한 번도 안 해봤다" "실무 개발자"까지 순서대로. 단계마다 뭘·왜 배우는지.

핵심 원칙: 외우지 말고 "왜 생겼는지" 흐름으로 이해. 모든 기술은 문제 해결의 결과다. 그리고 입력만 말고 직접 만들어봐야 실력이 된다.

레벨 0 — 개념 잡기 입문

1
코딩 vs 프로그래밍
프로그램 = 컴퓨터가 할 일을 순서대로 적은 것(운동 프로그램처럼). 코딩 = 컴퓨터에 일 시키기. 코드 = 그 수단.
쉽게: 컴퓨터한테 "이거 해, 그다음 저거 해" 하고 적어주는 게 프로그래밍.
2
인터넷 vs 웹
인터넷 = 전 세계 컴퓨터를 잇는 도로망. 웹 = 그 도로로 문서(소포)를 주소(URL)에 배달하는 우편 시스템.
예문: "네이버 접속" = 내 컴퓨터가 인터넷(도로)을 통해 네이버 서버에 "그 페이지 줘" 요청 서버가 HTML 배달.

레벨 1 — 웹의 두 기둥 입문

3
프론트엔드 — 사용자 눈에 보이는 화면 (HTML·CSS·JavaScript)
4
백엔드 — 화면 뒤에서 데이터 처리하는 엔진 (서버·DB·API)
쉽게: 식당으로 치면 프론트엔드=홀(손님이 보는 곳), 백엔드=주방(요리·재료 관리). 둘이 맞물려 서비스가 굴러간다.

레벨 2 — 연결과 저장 초중급

5
API / REST — 프론트와 백엔드가 데이터 주고받는 약속(메뉴판). CRUD로 동작.
6
데이터베이스(DB) — 데이터를 구조적으로 저장(집=DB, 방=테이블). SQL로 다룸.
7
클라우드 / 배포 — 만든 걸 인터넷에 올려 남도 쓰게(서버·IP·도메인·DNS·도커).

레벨 3 — 실무 역량 중급

신입 백엔드 최소 기준 4가지: ① REST API 설계(게시판 CRUD) ② DB/SQL ③ 클라우드 배포 ④ 전반적 웹 흐름 이해(웹서버·WAS·DB 관계를 그림 한 장으로). + 디버깅·Git·자료구조.
8
자료구조 & 설계 — 스택·큐·해시·연결리스트. "데이터 구조 제대로 고르면 알고리즘은 자연히 풀린다"(파이크 규칙).
9
서비스 라이프사이클 — 기획(MVP)설계개발테스트배포운영수익화. 수익화는 기획부터 고민.

레벨 4 — AI 시대 개발자 심화

10
AI·LLM 이해 — 머신러닝딥러닝트랜스포머LLM. 토큰·환각·RAG ( AI 탭)
11
AI 도구 활용 — 클로드·코덱스로 바이브코딩, 헤르메스/하네스로 에이전트 ( AI 도구 탭)
12
에이전틱 엔지니어 — 코드 짜는 사람 AI를 설계·오케스트레이션하는 사람. 1인이 팀급 생산성.
"이해 먼저, AI는 그다음." 기본기를 알아야 AI에게 정확히 지시하고, AI가 틀린 걸 잡아낼 수 있다.
