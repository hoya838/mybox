PostgreSQL + SQL DB · 백엔드 영역

데이터를 구조적으로 저장. 백엔드가 다룸.

1
기초 설치, 테이블 생성, CRUD(SELECT·INSERT·UPDATE·DELETE), 자료형
2
중급 JOIN, 인덱스, 제약조건(PK/FK), 집계(GROUP BY), 트랜잭션
3
심화 쿼리 최적화(EXPLAIN), 정규화, 뷰·함수, 동시성, 백업
CREATE TABLE todos (
  id SERIAL PRIMARY KEY,
  title TEXT NOT NULL,
  done BOOLEAN DEFAULT false
);
SELECT * FROM todos WHERE done = false;   -- 안 끝난 일
만드는 것: 데이터 창고. FastAPI·Express가 여기 연결해 진짜 저장.

핵심 개념

1. 테이블 생성 (자료형)

CREATE TABLE todos (
  id      SERIAL PRIMARY KEY,   -- 자동 증가 번호
  title   TEXT NOT NULL,        -- 빈 값 금지
  done    BOOLEAN DEFAULT false,
  created TIMESTAMP DEFAULT now()
);
쉽게: 엑셀 시트 만들기. 칸(컬럼)마다 타입·규칙 지정. PK=중복 없는 대표키.

2. CRUD — 4대 명령

INSERT INTO todos (title) VALUES ('공부');     -- 생성
SELECT * FROM todos WHERE done = false;        -- 조회
UPDATE todos SET done = true WHERE id = 1;     -- 수정
DELETE FROM todos WHERE id = 1;                -- 삭제
쉽게: SELECT=꺼내, INSERT=넣어, UPDATE=고쳐, DELETE=지워. REST의 CRUD와 1:1.

3. JOIN — 테이블 연결

SELECT todos.title, users.name
FROM todos
JOIN users ON todos.user_id = users.id;
쉽게: 흩어진 표를 공통 키로 합쳐 보기. "할일 + 그 주인 이름" 한 번에.

4. 인덱스·트랜잭션

CREATE INDEX idx_done ON todos(done);   -- 검색 빠르게

BEGIN;                                  -- 트랜잭션 시작
UPDATE accounts SET bal = bal-100 WHERE id=1;
UPDATE accounts SET bal = bal+100 WHERE id=2;
COMMIT;                                 -- 둘 다 성공해야 확정
쉽게: 인덱스=책 색인(빨리 찾기). 트랜잭션=송금처럼 "전부 되거나 전부 취소"(중간 사고 방지).

필수 문법 총정리 — 한 파일, 줄별 주석

SQL은 "무엇을 원하는지" 선언하는 언어. 테이블 만들기부터 CRUD·JOIN·집계까지 한 흐름으로. 주석은 --(두 하이픈). 명령 끝은 ;.

-- ── 1. 테이블 생성: 칸(컬럼)과 규칙 정의 ─────────
CREATE TABLE users (            -- users 테이블 만들기
  id     SERIAL PRIMARY KEY,    -- SERIAL=자동증가, PRIMARY KEY=중복없는 대표키
  name   VARCHAR(50) NOT NULL,  -- 최대 50자 문자열, NOT NULL=빈 값 금지
  email  VARCHAR(100) UNIQUE    -- UNIQUE=같은 값 중복 금지
);
CREATE TABLE posts (
  id        SERIAL PRIMARY KEY,
  title     TEXT NOT NULL,      -- TEXT=길이 제한 없는 문자열
  user_id   INTEGER REFERENCES users(id),  -- 외래키: users.id를 가리킴(연결)
  created   TIMESTAMP DEFAULT now()         -- DEFAULT=값 안 주면 현재시각 자동
);

-- ── 2. CRUD: 생성·조회·수정·삭제 ────────────────
INSERT INTO users (name, email)          -- 어느 컬럼에
VALUES ('철수', 'chul@x.com');           -- 어떤 값을 넣을지. → 1행 생성

SELECT id, name FROM users               -- users에서 id,name 컬럼만 꺼내기
WHERE name = '철수'                       -- 조건: 이름이 철수인 행만
ORDER BY id DESC                          -- id 내림차순 정렬(큰 게 위)
LIMIT 10;                                 -- 최대 10개만

UPDATE users SET email = 'new@x.com'     -- email 값을 바꿈
WHERE id = 1;                            -- id=1 행만(WHERE 없으면 전체 변경 — 위험!)

DELETE FROM users WHERE id = 1;          -- id=1 행 삭제(WHERE 필수)

-- ── 3. JOIN: 여러 테이블 합쳐 보기 ──────────────
SELECT posts.title, users.name           -- 두 테이블의 컬럼을 함께
FROM posts                               -- 기준 테이블
JOIN users ON posts.user_id = users.id;  -- 연결 조건: 외래키=대표키가 같은 행끼리

-- ── 4. 집계: 그룹으로 묶어 계산 ─────────────────
SELECT user_id, COUNT(*) AS cnt          -- user_id별로, 행 개수를 cnt라는 이름으로
FROM posts
GROUP BY user_id                         -- user_id가 같은 것끼리 묶기
HAVING COUNT(*) >= 2;                     -- 묶은 결과 중 2개 이상인 그룹만(WHERE의 집계판)

-- ── 5. 트랜잭션: 전부 성공 or 전부 취소 ─────────
BEGIN;                                    -- 트랜잭션 시작
UPDATE accounts SET bal = bal - 100 WHERE id = 1;  -- 출금
UPDATE accounts SET bal = bal + 100 WHERE id = 2;  -- 입금
COMMIT;                                   -- 둘 다 성공 시 확정(하나 실패면 ROLLBACK)
꼭 알아야 할 지식: ①WHERE 없는 UPDATE·DELETE = 전체 행 변경/삭제 — 사고 1순위, 항상 확인 ②PRIMARY KEY=행 고유식별, REFERENCES(외래키)=테이블 연결 ③WHERE=행 필터, HAVING=그룹 필터(다름) ④인덱스는 자주 WHERE·JOIN하는 컬럼에 — 조회 빠르지만 쓰기 느려짐 ⑤NULL 비교는 = NULL 아니라 IS NULL ⑥돈·재고처럼 여러 줄이 함께 바뀌면 트랜잭션으로 묶기.
