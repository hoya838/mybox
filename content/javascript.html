JavaScript 프론트 + 백(Node)

웹 공용어. 화면 동작 + (Node로) 서버까지.

왜 배우나: 바이브코딩 시대에도 문법을 배우는 이유는 AI 학습 5요소 지도 참고 — 도메인지식(1층 재료)이 곧 AI 결과물 검증 능력이다.

핵심 개념

1. 변수 — let / const

const name = "철수";   // 안 바뀜 (기본)
let age = 20;          // 바뀔 수 있음
age = 21;              // OK
// const는 재할당 불가
쉽게: const 먼저 쓰고, 바뀌어야 할 때만 let. var는 옛날 거라 안 씀.

2. 함수 — 일 묶음

function add(a, b) { return a + b; }
const add2 = (a, b) => a + b;   // 화살표 함수(요즘 스타일)
add(2, 3);   // 5
쉽게: 자판기. 입력(재료) 넣으면 출력(결과) 나옴. =>는 함수 짧게 쓰는 법.

3. 배열·객체 — 데이터 담기

const fruits = ["사과", "배"];        // 배열(순서)
fruits.push("귤");                    // 추가
const user = { name: "철수", age: 20 };  // 객체(이름표-값)
user.name;   // "철수"
fruits.map(f => f + "!");   // 각 요소 변형 → ["사과!",...]
쉽게: 배열=번호 매긴 줄, 객체=이름표 붙인 서랍. map=전부 하나씩 바꾸기.

4. 비동기 — 기다리는 일

서버 호출처럼 시간 걸리는 일은 기다렸다 처리. async/await.

async function load() {
  const res = await fetch('/api/todos');  // 응답 올 때까지 기다림
  const data = await res.json();          // JSON 변환
  console.log(data);
}
쉽게: 음식 주문 후 "나오면 받기". await=나올 때까지 기다려. 안 기다리면 빈손으로 다음 줄 감.

5. DOM·이벤트 — 화면 조작

const btn = document.querySelector('button');  // 요소 찾기
btn.addEventListener('click', () => {          // 클릭 시
  btn.textContent = '눌림!';                    // 내용 바꾸기
});
쉽게: "이 버튼 찾아서 → 누르면 → 이거 해". 화면이 살아 움직이게.

6. 조건·반복

if (age >= 20) { ... } else { ... }
for (const f of fruits) { console.log(f); }   // 하나씩
fruits.forEach(f => console.log(f));          // 같은 일
쉽게: if=갈림길, for=반복. 배열은 forEach/map으로도 돌림.

필수 문법 총정리 — 한 파일, 줄별 주석

아래 한 덩어리에 변수·자료형·함수·조건·반복·배열·객체·비동기가 다 들어있다. 주석은 그 줄이 하는 일 + 어떻게 읽는지를 설명한다.

// ── 1. 출력과 주석 ──────────────────────────────
console.log("Hello");   // console.log(x) = x를 콘솔에 찍어라. 디버깅 기본
// 이 줄은 //로 시작 → 실행 안 됨(사람 메모)

// ── 2. 변수: const(불변) / let(가변) ────────────
const PI = 3.14;        // const = 한 번 정하면 재할당 금지. "고정값"으로 읽기
let count = 0;          // let = 나중에 바뀔 값. "변할 상자"로 읽기
count = count + 1;      // count에 1 더해 다시 넣음 → 1. const였다면 여기서 에러

// ── 3. 자료형 6가지 ─────────────────────────────
const num = 42;         // 숫자(정수·소수 구분 없음)
const str = "글자";      // 문자열(따옴표)
const bool = true;      // 불리언(참/거짓) — 조건에 씀
const nothing = null;   // "값이 비었음"을 개발자가 명시
let undef;              // 값 안 넣으면 undefined("아직 없음")
const arr = [1, 2, 3];  // 배열(객체의 일종)

// ── 4. 템플릿 문자열: 변수 끼워넣기 ──────────────
const name = "철수";
const msg = `안녕 ${name}, ${num}살`;  // 백틱`` 안에서 ${ } = 값 삽입. "안녕 철수, 42살"

// ── 5. 연산자 ───────────────────────────────────
// 산술 연산자: 숫자 계산
const add1 = 3 + 4;     // + 더하기 → 7
const sub = 7 - 2;      // - 빼기 → 5
const mul1 = 3 * 4;     // * 곱하기 → 12
const div = 10 / 3;     // / 나누기 → 3.333...
const mod = 10 % 3;     // % 나머지(모듈로) → 1. 짝/홀 판별 등에 자주 씀
const sum = 3 + 4;      // 산술 종합 예시

// 대입 연산자: 변수에 값 넣기/누적
let count = 0;
count += 1;             // += 기존값에 더해서 재대입 → count = count + 1
count -= 1;             // -= 기존값에서 빼서 재대입
count *= 2;              // *= 기존값에 곱해서 재대입

// 비교 연산자: 참/거짓 판정
const same = (3 === 3); // === 값+타입 둘 다 같나? true. (== 는 쓰지 마라, 타입 자동변환 버그 유발)
const diff = (3 !== 4); // !== 값 또는 타입이 다른가? true
const gt = (5 > 3);     // > 왼쪽이 크다 → true
const lt = (5 < 3);     // < 왼쪽이 작다 → false
const gte = (5 >= 5);   // >= 크거나 같다 → true
const lte = (5 <= 4);   // <= 작거나 같다 → false

// 논리 연산자: 조건 조합
const both = true && false;  // &&(그리고, AND) 둘 다 참이어야 참 → false
const either = true || false; // ||(또는, OR) 하나만 참이면 참 → true
const not = !true;      // !(부정, NOT) 참/거짓 뒤집기 → false

// 증감 연산자: 1씩 늘리기/줄이기
let n = 1;
n++;                     // ++ 1 증가(후위) → n은 2
n--;                     // -- 1 감소 → n은 다시 1

// ── 6. 조건문: 갈림길 ───────────────────────────
if (num >= 20) {        // 괄호 조건이 참이면 아래 블록 실행
  console.log("성인");
} else {                // 거짓이면 이쪽
  console.log("미성년");
}

// ── 7. 반복문: 같은 일 여러 번 ──────────────────
for (let i = 0; i < 3; i++) {  // i=0에서 시작, 3보다 작은 동안, 매번 +1 → 0,1,2
  console.log(i);
}

// ── 8. 함수: 입력→출력 기계 ─────────────────────
function add(a, b) {    // a,b = 매개변수(받을 재료)
  return a + b;         // return = 결과를 돌려줌. 이 줄에서 함수 끝
}
const mul = (a, b) => a * b;   // 화살표 함수. => 뒤가 곧 return값(요즘 스타일)
const greet = (n = "손님") => `${n}님`;  // n=기본값. 안 넘기면 "손님"
add(2, 3);              // 함수 호출 → 5

// ── 9. 배열 메서드: 반복을 짧게 ─────────────────
const nums = [1, 2, 3];
nums.push(4);           // 끝에 추가 → [1,2,3,4]
const doubled = nums.map(n => n * 2);      // 각 요소 변형 → [2,4,6,8]
const evens = nums.filter(n => n % 2 === 0); // 조건 참인 것만 → [2,4]
const total = nums.reduce((acc, n) => acc + n, 0); // 하나로 합침 → 10

// ── 10. 객체: 이름표-값 묶음 ────────────────────
const user = { name: "철수", age: 20 };  // 키:값 쌍
user.age = 21;          // 점 표기로 값 변경
const { name: userName } = user;  // 구조분해: user.name을 userName으로 꺼냄
const copy = { ...user, city: "서울" };  // ...=펼치기(복사+추가)

// ── 11. 비동기: 시간 걸리는 일 ──────────────────
async function load() {           // async = 이 안에서 await 쓸 수 있음
  try {                           // try = 실패할 수 있는 코드 감싸기
    const res = await fetch("/api"); // await = 응답 올 때까지 멈춰서 기다림
    const data = await res.json();   // 본문을 JSON 객체로 변환(이것도 대기)
    return data;
  } catch (err) {                 // 위에서 에러 나면 여기로 점프
    console.error("실패:", err);
  }
}

// ── 12. 모듈: 파일 나눠 쓰기 ────────────────────

꼭 알아야 할 지식: ①== 금지, === 사용(타입까지 비교) ②변수는 const 먼저, 바뀔 때만 let, var는 안 씀 ③배열·객체는 참조로 전달 — 복사하려면 ... 스프레드 ④await는 async 함수 안에서만 ⑤null=의도적 빈값, undefined=아직 없음 ⑥map/filter/reduce가 for보다 실무 표준.
