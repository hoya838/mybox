Next.js 프론트 (React 프레임워크)

React 기반 풀 프레임워크. 라우팅·서버렌더·API까지 한 틀에.

선수: JavaScript → React 먼저. Next.js = React에 라우팅·SSR·API를 얹은 것. 아래 React 기초가 곧 Next.js 기초.
Next.js가 React에 더해주는 것
기능뜻
파일 기반 라우팅app/about/page.tsx = /about 자동 라우트
서버 컴포넌트기본이 서버 렌더 — 빠르고 SEO 좋음
SSR / SSG서버에서 미리 그려 보냄 / 빌드 때 정적 생성
API Routesapp/api/todos/route.ts로 백엔드 API도 같은 프로젝트에
최적화 내장<Image>·폰트·번들 자동 최적화
쉽게: React = 화면 조립 부품. Next.js = 그 부품 + "주소 자동 연결 + 서버에서 미리 그리기 + API까지" 다 갖춘 완성 키트. 실무 웹은 대부분 Next.js.

핵심 개념

1. 컴포넌트 + JSX

화면 조각을 함수로. HTML 같은 JSX를 반환.

function Hello({ name }) {     // props로 값 받기
  return <h1>안녕 {name}</h1>;  // JSX (HTML+JS)
}
<Hello name="철수" />          // 사용
쉽게: 레고 블록 하나 = 컴포넌트. { } 안엔 JS 값. 재사용 가능.

2. state — 바뀌는 값(useState)

const [count, setCount] = useState(0);  // [값, 바꾸는함수]
<button onClick={() => setCount(count+1)}>
  {count}
</button>
쉽게: state 바뀌면 화면 자동 다시 그림. setCount로만 바꿔야 React가 앎.

3. useEffect — 부수 작업

useEffect(() => {
  fetch('/api/todos').then(r=>r.json()).then(setTodos);
}, []);   // [] = 처음 1번만 실행
쉽게: "화면 뜨면 데이터 불러와" 같은 곁다리 작업. []=언제 실행할지.

4. 리스트 렌더

{todos.map(t => (
  <li key={t.id}>{t.title}</li>   // key 필수
))}
쉽게: 배열을 map으로 화면 요소들로. key=각 항목 구분표(React가 추적).

Next.js 핵심 (React 위에)

1. 파일 기반 라우팅 (App Router)

폴더 구조가 곧 URL. 라우터 설정 코드 없음.

app/
 ├ page.tsx          → /
 ├ about/page.tsx    → /about
 └ todos/[id]/page.tsx → /todos/3  (동적)
쉽게: 폴더 만들면 그게 주소. [id]=바뀌는 부분(상세페이지).

2. 서버 컴포넌트 (기본)

// app/todos/page.tsx — 서버에서 실행
async function Page() {
  const todos = await db.query('SELECT * FROM todos'); // DB 직접!
  return <ul>{todos.map(t => <li>{t.title}</li>)}</ul>;
}
쉽게: 컴포넌트가 서버에서 돌아 DB를 직접 불러옴 → 빠르고 SEO 좋음. 클릭 동작 필요하면 'use client' 붙임.

3. API Routes (백엔드 내장)

// app/api/todos/route.ts
export async function GET() {
  return Response.json([{ title: '공부' }]);
}
export async function POST(req) {
  const body = await req.json(); ...
}
쉽게: 프론트 프로젝트 안에 백엔드 API까지. 작은 서비스는 Next.js 하나로 끝(FastAPI·Express 없이도).

4. 시작하기

npx create-next-app@latest my-app
cd my-app && npm run dev   # localhost:3000
학습 순서: JavaScript → React(컴포넌트·state·useEffect 위 기초) → Next.js(라우팅·서버컴포넌트·API). 백엔드를 Python(FastAPI)/Node(Express)로 따로 둘지, Next API로 합칠지는 규모 따라.
