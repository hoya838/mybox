Next.js 프론트 (React 프레임워크)

React 기반 풀 프레임워크. 라우팅·서버렌더·API까지 한 틀에.

선수: JavaScript → React 먼저. Next.js = React에 라우팅·SSR·API를 얹은 것. 아래 React 기초가 곧 Next.js 기초.
Next.js가 React에 더해주는 것
기능뜻
파일 기반 라우팅app/about/page.tsx = /about 자동 라우트
서버 컴포넌트기본이 서버 렌더 — 빠르고 SEO 좋음
SSR / SSG서버에서 미리 그려 보냄 / 빌드 때 정적 생성
API Routesapp/api/todos/route.ts로 백엔드 API도 같은 프로젝트에
최적화 내장<Image>·폰트·번들 자동 최적화
쉽게: React = 화면 조립 부품. Next.js = 그 부품 + "주소 자동 연결 + 서버에서 미리 그리기 + API까지" 다 갖춘 완성 키트. 실무 웹은 대부분 Next.js.

핵심 개념

1. 컴포넌트 + JSX

화면 조각을 함수로. HTML 같은 JSX를 반환.

function Hello({ name }) {     // props로 값 받기
  return <h1>안녕 {name}</h1>;  // JSX (HTML+JS)
}
<Hello name="철수" />          // 사용
쉽게: 레고 블록 하나 = 컴포넌트. { } 안엔 JS 값. 재사용 가능.

2. state — 바뀌는 값(useState)

const [count, setCount] = useState(0);  // [값, 바꾸는함수]
<button onClick={() => setCount(count+1)}>
  {count}
</button>
쉽게: state 바뀌면 화면 자동 다시 그림. setCount로만 바꿔야 React가 앎.

3. useEffect — 부수 작업

useEffect(() => {
  fetch('/api/todos').then(r=>r.json()).then(setTodos);
}, []);   // [] = 처음 1번만 실행
쉽게: "화면 뜨면 데이터 불러와" 같은 곁다리 작업. []=언제 실행할지.

4. 리스트 렌더

{todos.map(t => (
  <li key={t.id}>{t.title}</li>   // key 필수
))}
쉽게: 배열을 map으로 화면 요소들로. key=각 항목 구분표(React가 추적).

Next.js 핵심 (React 위에)

1. 파일 기반 라우팅 (App Router)

폴더 구조가 곧 URL. 라우터 설정 코드 없음.

app/
 ├ page.tsx          → /
 ├ about/page.tsx    → /about
 └ todos/[id]/page.tsx → /todos/3  (동적)
쉽게: 폴더 만들면 그게 주소. [id]=바뀌는 부분(상세페이지).

2. 서버 컴포넌트 (기본)

// app/todos/page.tsx — 서버에서 실행
async function Page() {
  const todos = await db.query('SELECT * FROM todos'); // DB 직접!
  return <ul>{todos.map(t => <li>{t.title}</li>)}</ul>;
}
쉽게: 컴포넌트가 서버에서 돌아 DB를 직접 불러옴 → 빠르고 SEO 좋음. 클릭 동작 필요하면 'use client' 붙임.

3. API Routes (백엔드 내장)

// app/api/todos/route.ts
export async function GET() {
  return Response.json([{ title: '공부' }]);
}
export async function POST(req) {
  const body = await req.json(); ...
}
쉽게: 프론트 프로젝트 안에 백엔드 API까지. 작은 서비스는 Next.js 하나로 끝(FastAPI·Express 없이도).

4. 시작하기

npx create-next-app@latest my-app
cd my-app && npm run dev   # localhost:3000
학습 순서: JavaScript → React(컴포넌트·state·useEffect 위 기초) → Next.js(라우팅·서버컴포넌트·API). 백엔드를 Python(FastAPI)/Node(Express)로 따로 둘지, Next API로 합칠지는 규모 따라.

필수 패턴 — 줄별 주석 (App Router)

Next.js는 폴더 구조가 곧 라우팅이다. 파일 위치·이름이 URL·역할을 정한다. 주석 = 그 줄/파일이 하는 일.

app/
├─ layout.tsx        // 모든 페이지 공통 껍데기(헤더·푸터). 필수
├─ page.tsx          // "/" 경로 화면
├─ posts/
│  ├─ page.tsx       // "/posts" 목록 화면
│  └─ [id]/page.tsx  // "/posts/3" 상세. [id]=동적 구간(폴더명이 변수)
└─ api/hello/route.ts // "/api/hello" 백엔드 엔드포인트
// ── 서버 컴포넌트 (기본값): 서버에서 실행, DB·API 직접 호출 ──
// app/posts/page.tsx  — "use client" 없으면 서버 컴포넌트
async function PostsPage() {          // async 가능 — 서버라서
  const res = await fetch("https://api.../posts", { cache: "no-store" });
  //  cache:"no-store" = 매 요청 최신값(캐시 안 함)
  const posts = await res.json();     // 서버에서 데이터 준비
  return (                            // 완성된 HTML을 브라우저로 전송
    <ul>
      {posts.map((p) => (             // 배열을 <li> 목록으로
        <li key={p.id}>{p.title}</li> // key=React가 항목 구분(필수)
      ))}
    </ul>
  );
}
export default PostsPage;             // 이 파일의 기본 내보내기 = 그 경로 화면

// ── 클라이언트 컴포넌트: 브라우저에서 실행, 상태·이벤트 필요할 때 ──
"use client";                         // 이 지시어가 있어야 useState·onClick 가능
import { useState } from "react";
function LikeButton() {
  const [n, setN] = useState(0);      // 상태 = 바뀌면 화면 자동 갱신
  return <button onClick={() => setN(n + 1)}>{n}</button>; // 클릭 시 +1
}

// ── 동적 라우트: URL의 값 꺼내기 ──────────────────
// app/posts/[id]/page.tsx
async function PostDetail({ params }) {  // params에 URL 구간이 담김
  const { id } = await params;           // "/posts/3" → id="3"
  const post = await fetch(`https://api.../posts/${id}`).then(r => r.json());
  return <h1>{post.title}</h1>;
}

// ── API 라우트: Next 안에 백엔드 ──────────────────
// app/api/hello/route.ts
export async function GET() {         // GET 요청 처리 함수(POST면 export POST)
  return Response.json({ msg: "hi" }); // JSON 응답 반환
}
꼭 알아야 할 지식: ①기본은 서버 컴포넌트(빠름·SEO·DB직접) — 상태/이벤트 필요할 때만 "use client" ②폴더명 [id]=동적, 파일 page.tsx=화면·route.ts=API·layout.tsx=공통틀 ③fetch 캐시 옵션(no-store=항상 최신, 기본=캐시)이 렌더 방식 결정 ④클라이언트 컴포넌트는 최대한 작게(잎사귀만) ⑤데이터 바꾼 뒤 router.refresh()로 서버 데이터 다시 당김.
