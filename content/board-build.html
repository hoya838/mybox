1. 데이터 흐름

요청 하나가 4개 층을 지난다. 이 순서를 먼저 그리고 각 층을 채웠다.

브라우저          Next.js              FastAPI            PostgreSQL
 (Client)   →    (Frontend)    →     (Backend)    →      (Data)
사용자 조작      서버컴포넌트 렌더      검증·CRUD·async      posts 테이블
             ← HTML·JSON ←        ← REST /posts ←     ← asyncpg SQL ←
쉽게: 목록·상세는 Next.js 서버 컴포넌트가 FastAPI를 직접 호출(cache: "no-store")해 매 요청 최신값 렌더. 작성·수정·삭제는 클라이언트 컴포넌트가 브라우저 fetch로 처리한 뒤 router.refresh()로 서버 데이터를 다시 당김.

2. 스택 선택 이유

주어진 4개 기술을 왜 이렇게 엮었는지.

기술역할선택 근거
Next.js App Router화면·라우팅파일 기반 라우팅(app/posts/[id])으로 목록·상세·작성·수정 경로를 폴더로 표현. 서버 컴포넌트에서 백엔드를 직접 호출해 데이터 로딩 코드를 줄임.
TypeScript타입 안전lib/api.ts에 Post·PostInput 타입을 한 곳에 정의 → 프론트 전 구간이 같은 계약 공유. 필드 오타·누락을 컴파일 시점에 차단.
FastAPI asyncAPI·검증Pydantic이 요청 본문을 자동 검증하고 /docs 문서를 자동 생성. 비동기(async def + asyncpg)로 DB 대기 중 다른 요청 처리.
PostgreSQL영속 저장관계형 표준. SQLAlchemy 2.0 ORM으로 Post 모델을 파이썬 클래스로 다루고, 실행은 asyncpg 드라이버가 담당.

3. 빌드 순서

데이터가 흐르는 반대 방향 — 저장소 → API → 화면 순으로 쌓았다. 아래층이 위층의 계약을 정하기 때문.

1
인프라 DB 컨테이너 — PostgreSQL 16을 docker compose로 정의(계정·DB명·헬스체크). 재현 가능한 DB부터. docker-compose.yml
2
백엔드 모델·연결 — 비동기 엔진/세션(database.py) + Post 테이블(models.py: id·title·author·content·created_at·updated_at). 설정은 env로 분리. config.py · database.py · models.py
3
백엔드 입출력 계약 — Pydantic PostCreate·PostUpdate·PostRead·PostList. 목록은 본문 제외 경량, 상세는 전체. schemas.py
4
백엔드 CRUD 라우트 — 5개 엔드포인트(GET 목록/상세·POST·PUT·DELETE). 없는 글 404, 생성 201, CORS 허용, 기동 시 테이블 자동생성. main.py
5
프론트 API 클라이언트 — 백엔드 계약을 TS 타입으로 옮기고 fetch 래퍼 5종 + 에러 처리 + 날짜 포맷터. 이후 모든 화면이 이 파일만 호출. lib/api.ts
6
프론트 레이아웃·목록 — 공통 상단바 + 스타일. 홈은 서버 컴포넌트로 목록을 서버에서 렌더. layout.tsx · globals.css · page.tsx
7
프론트 상세·작성·수정·삭제 — 상세는 서버 컴포넌트, 폼·삭제 버튼은 상태·이벤트가 필요해 클라이언트 컴포넌트로 분리. 작성/수정이 폼 하나 공유(mode 분기). posts/[id] · posts/new · PostForm.tsx
8
점검 검증 — 백엔드 앱 임포트로 라우트 확인 + tsc --noEmit 타입체크 + E2E CRUD 실행. tsc · uvicorn import · README.md

6. 실행 방법

DB·백엔드·프론트 3개 프로세스. npm run dev는 frontend/에서, uvicorn은 backend/에서.

1) PostgreSQL 기동

cd 게시판 && docker compose up -d          # localhost:5432

2) 백엔드 (localhost:8000, 문서 /docs)

cd backend && python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt && cp .env.example .env
uvicorn app.main:app --reload

3) 프론트 (localhost:3000)

cd frontend && npm install && cp .env.local.example .env.local
npm run dev
