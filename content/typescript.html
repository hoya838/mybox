TypeScript 프론트 + 백

JavaScript + 타입 안전장치. 큰 프로젝트 필수.

관점: "타입은 코드의 채점표" — 왜 그런지는 AI 학습 5요소 지도의 4층 판정 구조에서. 채점 기준이 있어야 개선이 가능하다는 명제가 타입 시스템에도 그대로 적용된다.
선수: JavaScript 기초 먼저(이건 JS에 타입만 더한 것).

기초 문법 — 순서대로

#배울 것예시
1기본 타입: string · number · boolean
2배열·튜플number[] · [string, number]
3함수 타입(a:number)=>number
4인터페이스/타입interface User{name:string}
5유니온·옵셔널number|string · age?:number
6제네릭function f<T>(x:T):T
7유틸리티 타입Partial<T> · Pick<T,K>
1
기초 타입 표기(string·number·boolean), 인터페이스, 함수 타입
2
중급 제네릭, 유니온·옵셔널, 타입 추론, 유틸리티 타입
3
심화 타입 가드, 고급 제네릭, 타입 안전 API, tsconfig 설계
interface Todo { id: number; title: string; done: boolean; }
function toggle(t: Todo): Todo {
  return { ...t, done: !t.done };   // 타입 안 맞으면 미리 에러
}
만드는 것: 오타·타입 실수가 실행 전에 잡히는 안전한 코드.

핵심 개념

1. 타입 표기

let name: string = "철수";
let age: number = 20;
let ok: boolean = true;
let nums: number[] = [1, 2, 3];     // 숫자 배열
function add(a: number, b: number): number { return a+b; }
쉽게: 변수 옆에 "이건 숫자야" 라벨. 문자열 넣으면 코딩 중 빨간 줄로 경고.

2. 인터페이스 — 객체 모양 정의

interface User {
  name: string;
  age: number;
  email?: string;   // ? = 있어도 없어도 됨(옵셔널)
}
const u: User = { name: "철수", age: 20 };
쉽게: "User는 이런 칸을 가져야 해" 설계도. 칸 빠지면 에러.

3. 유니온 — 여러 타입 허용

let id: number | string;   // 둘 중 하나
id = 1;     // OK
id = "a1";  // OK
type Status = "active" | "done";  // 정해진 값만
쉽게: |=또는. "숫자 또는 문자" 식으로 허용 범위 지정.

4. 제네릭 — 타입을 변수처럼

function first<T>(arr: T[]): T { return arr[0]; }
first([1,2,3]);       // number
first(["a","b"]);     // string
쉽게: T=나중에 정해질 타입 자리. 한 함수로 여러 타입 안전하게.
