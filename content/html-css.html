HTML · CSS 프론트엔드

화면의 뼈대(HTML)와 스타일(CSS). 모든 웹의 시작.

핵심 개념

1. 태그 = 의미 있는 상자

HTML은 내용을 태그로 감싸 뜻을 부여. 여는 태그 <p> + 닫는 태그 </p> 짝.

<h1>큰 제목</h1>
<p>문단입니다</p>
<ul><li>목록 항목</li></ul>
<a href="https://naver.com">링크</a>
<img src="cat.jpg" alt="고양이">
쉽게: 글에 형광펜으로 "여긴 제목, 여긴 문단" 표시하는 것. 의미대로 골라 쓰기(시맨틱).

2. 박스 모델 — 모든 요소는 상자

각 요소 = 내용 + padding(안쪽 여백) + border(테두리) + margin(바깥 여백).

.box {
  padding: 16px;        /* 내용~테두리 사이 */
  border: 1px solid #ccc;
  margin: 24px;         /* 상자~상자 사이 */
}
쉽게: 선물상자 = 물건(내용) + 포장지 안 완충재(padding) + 상자(border) + 상자끼리 거리(margin).

3. 선택자 — 누구를 꾸밀까

h1        { color: blue; }     /* 모든 h1 */
.btn      { ... }              /* class="btn" */
#header   { ... }              /* id="header" */
.btn:hover{ ... }              /* btn에 마우스 올렸을 때 */
쉽게: .=클래스(여러 개), #=아이디(딱 하나), :hover=상태.

4. 레이아웃 — Flexbox

요소를 가로·세로로 정렬하는 현대 표준.

.row {
  display: flex;            /* 가로 배치 */
  justify-content: space-between;  /* 양끝 정렬 */
  align-items: center;      /* 세로 가운데 */
  gap: 12px;
}
쉽게: 컨테이너에 display:flex 주면 자식들이 줄 맞춰 섬. 정렬은 옵션으로.

5. 반응형 — 화면 크기 대응

@media (max-width: 600px) {
  .menu { display: none; }   /* 폰에선 숨김 */
  .row  { flex-direction: column; }  /* 세로 쌓기 */
}

필수 문법 총정리 — 줄별 주석

HTML=구조, CSS=꾸미기. 둘은 언어라기보다 표기법이라 "문법"이 곧 태그·속성이다. 주석 표기가 서로 다르다: HTML <!-- -->, CSS /* */.

HTML — 페이지 뼈대

<!-- HTML 주석은 이렇게. 화면에 안 보임 -->
<!DOCTYPE html>              <!-- "이 문서는 HTML5" 선언(맨 첫 줄) -->
<html lang="ko">            <!-- 문서 루트. lang=언어(검색·번역용) -->
<head>                      <!-- 화면 밖 정보(제목·연결파일) -->
  <meta charset="UTF-8">    <!-- 글자 인코딩. 한글 안 깨지게 필수 -->
  <title>내 페이지</title>   <!-- 브라우저 탭에 뜨는 제목 -->
</head>
<body>                      <!-- 실제 눈에 보이는 내용 전부 -->
  <h1>제목</h1>             <!-- 최상위 제목(페이지당 1개 권장) -->
  <p>문단 텍스트</p>         <!-- p=paragraph, 한 문단 -->
  <a href="https://x.com">링크</a>  <!-- href=이동할 주소. a=anchor -->
  <img src="cat.png" alt="고양이">  <!-- src=이미지 경로, alt=대체텍스트(접근성) -->
  <ul>                     <!-- 순서없는 목록(점) -->
    <li>항목1</li>         <!-- li=목록 항목 -->
  </ul>
  <button id="go" class="btn">클릭</button>  <!-- id=고유이름(1개), class=분류(여러개 공유) -->
  <input type="text" placeholder="입력">  <!-- 사용자 입력칸. type이 종류 결정 -->
</body>
</html>
읽는 법: <태그>내용</태그>가 기본 단위. 여는 태그·닫는 태그로 감싼다. id는 이름표(유일), class는 그룹표(공유) — CSS·JS가 이걸로 요소를 찾는다.

CSS — 그 뼈대를 꾸미기

/* CSS 주석은 이렇게 */
h1 {                    /* 선택자: 어떤 요소에 적용할지. 여기선 모든 h1 */
  color: #333;          /* 속성: 값 — 글자색(진회색). ;로 각 줄 끝냄 */
  font-size: 24px;      /* 글자 크기 */
}
.btn {                  /* .클래스이름 = class="btn"인 요소 전부 */
  background: #2563eb;  /* 배경색 */
  padding: 8px 16px;    /* 안쪽 여백: 위아래 8, 좌우 16 */
  border-radius: 6px;   /* 모서리 둥글게 */
}
#go {                   /* #아이디 = id="go"인 요소 하나 */
  display: none;        /* 화면에서 숨김 */
}
.row {
  display: flex;        /* 자식들을 가로로 나란히(Flexbox) */
  gap: 12px;            /* 자식 사이 간격 */
  justify-content: center;  /* 가로 가운데 정렬 */
}
@media (max-width: 600px) {  /* 화면 폭 600px 이하일 때만 아래 적용(반응형) */
  .row { flex-direction: column; }  /* 세로로 쌓기 */
}
꼭 알아야 할 지식: ①CSS 구조 = 선택자 { 속성: 값; } ②선택자 우선순위: #id > .class > 태그 ③박스모델 — 요소는 content→padding(안여백)→border→margin(바깥여백) 순 겹 ④레이아웃은 Flexbox(1차원)·Grid(2차원)가 현대 표준 ⑤@media로 화면 크기별 대응(반응형) ⑥HTML은 의미 태그(header·nav·main·footer)를 div보다 우선 — 접근성·SEO.
쉽게: "폰처럼 좁으면 이렇게 바꿔" 규칙. PC는 가로, 폰은 세로.
